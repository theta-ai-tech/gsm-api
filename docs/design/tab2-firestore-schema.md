# Firestore Schema — Tab 2 IMPROVE

> ⚠️ **Design / decision record — non-canonical.** This document captures intent and history and is *not* kept in lockstep with the code. For current behavior, defer to the canonical docs under [`../README.md`](../README.md).

This document describes the Firestore collections, sub-collections, and fields
introduced or extended by the Tab 2 (IMPROVE) feature. All field names in
Firestore use **camelCase**. Python model attributes use `snake_case` and are
mapped by the repo layer (`repos/mappers.py`).

---

## 1. `users/{uid}` — extended fields

The root user document gains two new top-level fields for Tab 2. Existing
fields (e.g. `name`, `email`, `rankings`, `upcomingMatches`, `completedMatches`)
are unchanged.

### `journalRecent` — `array<JournalEntrySummary>`

A denormalised cache of the user's **most-recent 10** journal entries, ordered
newest-first. Updated atomically (in a Firestore transaction) every time an
entry is created. Used by the home screen to render the journal preview without
querying the sub-collection.

Capped at **10 items** — older items are dropped on each write
(`app/constants.py:JOURNAL_RECENT_MAX = 10`).

| Field | Type | Description |
|-------|------|-------------|
| `entryId` | `string` | Document ID of the journal entry |
| `createdAt` | `timestamp` | Creation timestamp (UTC) |
| `title` | `string` | Entry title |
| `matchId` | `string \| null` | Linked match ID (if `entryType == "match"`) |
| `sport` | `string \| null` | Sport enum value (`"tennis"`, `"padel"`, `"pickleball"`) |
| `entryType` | `string \| null` | `"match"` or `"training"` |

### `northStarGoal` — `map<NorthStarGoal> | null`

The user's current North Star goal. Written atomically by `PUT /me/north-star`.
Always overwritten in full — no partial updates.

| Field | Type | Description |
|-------|------|-------------|
| `goalText` | `string` | Goal description (max 500 chars) |
| `progressPct` | `float` | Progress percentage (0.0–100.0); reset to `0.0` on every `PUT` |
| `createdAt` | `timestamp` | Timestamp of the most recent goal upsert |
| `targetDate` | `timestamp \| null` | Optional target date |

## 2. `users/{uid}/journalEntries/{entry_id}` — new sub-collection

Each document represents a single journal entry. The `entry_id` is
auto-generated by Firestore (`document()` — no client-chosen ID).

### Fields

| Field | Type | Nullable | Description |
|-------|------|----------|-------------|
| `createdAt` | `timestamp` | no | UTC creation time; used as the primary sort key |
| `title` | `string` | no | Entry title (max 200 chars) |
| `body` | `string` | no | Free-text notes (max 5 000 chars) |
| `tags` | `array<string>` | no | User-supplied tags, e.g. `["tournament", "grass"]` (max 20) |
| `visibility` | `string` | no | `"private"` or `"public"` |
| `entryType` | `string` | no | `"match"` or `"training"` |
| `sport` | `string` | yes | Sport enum value |
| `matchId` | `string` | yes | ID of the linked match document (for `entryType == "match"`) |
| `scoreText` | `string` | yes | Denormalised score string, e.g. `"6-4 7-5"` |
| `result` | `string` | yes | `"W"` or `"L"` (denormalised from match; set on create only) |
| `durationMinutes` | `int` | yes | Training duration in minutes (required when `entryType == "training"`, must be > 0) |
| `trainingFocus` | `array<string>` | no | Training focus tags, e.g. `["serve", "footwork"]`; empty for match entries |
| `reflection` | `map<MatchReflection>` | yes | Post-match reflection; `null` at creation, written via `PATCH` |
| `clientRequestId` | `string` | yes | Optional client idempotency key (scoped per user) |
| `isDeleted` | `boolean` | no | Soft-delete flag; defaults to `false` |
| `deletedAt` | `timestamp` | yes | Soft-delete timestamp; `null` when active |

### `reflection` sub-map (`MatchReflection`)

| Field | Type | Description |
|-------|------|-------------|
| `wentWell` | `array<string>` | Skill tags the player rated positively, e.g. `["first_serve", "net_play"]` |
| `wentWrong` | `array<string>` | Skill tags the player rated negatively, e.g. `["double_faults"]` |
| `opponentWeak` | `array<string>` | Opponent weakness tags (used for scouting pipeline) |
| `opponentStrong` | `array<string>` | Opponent strength tags |
| `aiSummary` | `string \| null` | Reserved for future AI-generated summary; always `null` currently |
| `reflectionVersion` | `string \| null` | Optional reflection taxonomy version (e.g. `"v1"`) |

## 4. Firestore indexes

### Composite index required for cursor-based pagination

Collection: `users/{uid}/journalEntries`

| Field | Order |
|-------|-------|
| `createdAt` | Descending |
| `__name__` (document ID) | Descending |

This two-field ordering allows `JournalRepo.list_entries` to implement
stable cursor pagination using `start_after([createdAt, doc_ref])`.

> **Note:** Firestore auto-indexes single fields. The composite index above
> must be deployed via `firestore.indexes.json` or the Firebase console before
> `GET /me/journal` will work in production.

## 5. Write patterns

| Operation | Firestore writes |
|-----------|-----------------|
| `POST /me/journal` | **Atomic transaction**: `set` on `journalEntries/{entry_id}` + `update journalRecent` on `users/{uid}` |
| `PATCH /me/journal/{entry_id}` | `update` on `journalEntries/{entry_id}` (partial — only non-null request fields) |
| `DELETE /me/journal/{entry_id}` | Soft delete: `update isDeleted/deletedAt` on `journalEntries/{entry_id}` + remove from `journalRecent` |
| `PUT /me/north-star` | `update northStarGoal` on `users/{uid}` (full map replacement) |
| `GET /me/journal` | Read-only query on `journalEntries` sub-collection |
| `GET /me/journal/{entry_id}` | Single-document `get` on `journalEntries/{entry_id}` |
| `GET /me/stats` | Single-document `get` on `users/{uid}` (compute-on-read from cached fields) |

Additional API behavior:
- Ownership mismatch for `GET/PATCH/DELETE /me/journal/{entry_id}` returns `403`.
- `POST /me/journal` is rate-limited per user (`JOURNAL_CREATE_RATE_LIMIT_PER_HOUR`, currently `50`/hour) and returns `429` on breach.
- `GET /me/journal` exposes query params `entry_type`, `sport`, `tag` for contract stability; currently unsupported filters return `400` with `filter not supported yet`.
